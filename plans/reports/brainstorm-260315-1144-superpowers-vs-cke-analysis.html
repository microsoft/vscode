3. Architecture Comparison

Superpowers Architecture

Skill Router Pattern: Injects ONE meta-skill (using-superpowers) at session start that forces agent to check/invoke skills before ANY action.

  • hooks/ — 1 hook (SessionStart), bash+polyglot
  • skills/ — 14 skills with SKILL.md + references
  • agents/ — 1 agent (code-reviewer)
  • commands/ — 3 commands (deprecated)
  • tests/ — 5 test suites

4. What Superpowers Does Better

Features CKE should learn from Superpowers

★★★ DOT Flowcharts as Executable Specs

GraphViz DOT diagrams in skills serve as the AUTHORITATIVE process definition. Models follow graphs more reliably than prose. Discovered "The Description Trap" — models follow short descriptions over detailed flowcharts.

CKE Action: Add DOT/Mermaid flows to cook, fix, brainstorm, plan

★★★ Hard Gates & Anti-Rationalization

Explicit <HARD-GATE> blocks prevent implementation before design. Comprehensive rationalization prevention tables catch LLM excuse patterns.

CKE Action: Add <HARD-GATE> blocks + rationalization tables to core skills

★★★ Verification Before Completion

Dedicated skill: "Evidence before claims, always." No "should work" — must run verification command and show output. Iron Law: NO COMPLETION CLAIMS WITHOUT FRESH VERIFICATION.

CKE Action: Integrate into code-review and test skills

★★ Two-Stage Code Review

Separates review into spec compliance (does it match requirements?) then code quality (is it well-written?). Catches well-written code that doesn't match spec.

CKE Action: Split code-review into two passes

★★ Subagent Context Isolation

"Subagents receive only the context they need." Each gets precisely crafted prompts — never session history. Preserves controller context.

CKE Action: Add context isolation guidelines to orchestration protocol

★★ Implementer Status Protocol

Structured statuses: DONE, DONE_WITH_CONCERNS, BLOCKED, NEEDS_CONTEXT. Controller handles each case appropriately.

CKE Action: Define standard status protocol for agent communication

★★ Git Worktree First-Class

Worktree REQUIRED before implementation. Auto-detect directory, verify .gitignore, run project setup, verify clean baseline.

CKE Action: Make worktree recommended in cook workflow

★★ Skill Test Infrastructure

5 test suites: skill-triggering, integration (claude -p), token analysis, e2e workflows, explicit request tests.

CKE Action: Build skill-triggering test suite

7. Key Learnings for CKE

Priority 1 Workflow Discipline (High Impact, Medium Effort)

  1. Add DOT/Mermaid process flows to cook, fix, brainstorm, plan as authoritative specs
  2. Add <HARD-GATE> blocks to prevent implementation before design approval
  3. Add anti-rationalization tables to cook, fix, test skills
  4. Integrate verification-before-completion into code-review and test

Priority 2 Quality Gates (High Impact, High Effort)

  1. Two-stage code review — spec-compliance + quality passes
  2. Implementer status protocol — DONE, DONE_WITH_CONCERNS, BLOCKED, NEEDS_CONTEXT
  3. Subagent context isolation — enforce minimal context in orchestration
  4. Scope assessment — detect over-scoped projects in brainstorm/plan

Priority 3 Testing & Quality (Medium Impact, High Effort)

  1. Skill-triggering tests — validate skills activate from naive prompts
  2. End-to-end workflow tests — brainstorm → plan → implement → review
  3. Token usage tracking — analyze cost per skill/workflow

Priority 4 Nice-to-Have

  1. Gemini CLI support — extension format + tool mapping
  2. Cursor plugin support — plugin manifest
  3. Brainstorm visual companion — WebSocket-based mockup viewer

8. Philosophy Comparison

Superpowers: "Discipline Through Enforcement"

  • Iron Laws that cannot be violated
  • Delete code written before tests
  • Skills as mandatory workflows
  • Every project goes through brainstorming
  • "Violating the letter = violating the spirit"
  • Heavy anti-rationalization focus

CKE: "Productivity Through Breadth"

  • YAGNI/KISS/DRY as principles
  • Skills as specialized tools
  • Agent orchestration for parallel execution
  • Rich ecosystem covering full lifecycle
  • Hooks for lifecycle management
  • Memory for cross-session continuity
The Ideal Framework combines Superpowers' discipline enforcement (hard gates, verification, anti-rationalization) with CKE's breadth and ecosystem (70+ skills, 14 agents, rich hooks, memory).

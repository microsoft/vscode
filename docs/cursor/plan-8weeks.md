# 8 周研发计划（Cursor 核心功能）

本计划以 VS Code 源码为基线，目标在 8 周内完成 Cursor 核心能力的底层架构与体验闭环。

## 第 1 周｜基线与架构落点（已完成）
- 目标：确定改造边界与核心服务入口
- 交付：
  - 架构落点与分层说明
  - 可执行的改造清单
  - `IAICoreService` 骨架与 `ChatService` 接入

## 第 2 周｜上下文系统 v1
- 目标：最小上下文构建闭环
- 交付：
  - `AICoreContextBuilder` 骨架
  - 当前文件 + 选区上下文注入
  - Chat 请求联动上下文构建

## 第 3 周｜上下文系统 v2 + Prompt 规范
- 目标：可扩展的上下文与提示词模板体系
- 交付：
  - 最近文件/工作区摘要
  - Prompt 模板与变量系统
  - 基础提示词版本管理

## 第 4 周｜索引与检索 v1
- 目标：跨文件检索与召回
- 交付：
  - 索引器、chunking、embedding 管道
  - 基础检索策略（TopK、过滤）
  - 与上下文融合

## 第 5 周｜智能编辑能力 v1
- 目标：可应用的改写与代码生成
- 交付：
  - Diff 预览与应用
  - 编辑确认/回滚策略
  - Chat/InlineChat 统一编辑出口

## 第 6 周｜Agent 工作流雏形
- 目标：多步任务执行与工具调用
- 交付：
  - 工具调度框架与安全确认
  - 任务链路 UI（状态、进度）
  - 最小 Agent 闭环

## 第 7 周｜性能与稳定性
- 目标：稳定、可持续使用
- 交付：
  - 缓存/节流/超时
  - 日志与诊断
  - 降级策略与失败兜底

## 第 8 周｜体验打磨与封装
- 目标：内部可用版本 + 迭代节奏
- 交付：
  - 设置面板与产品化入口
  - 打包与版本策略
  - 第二阶段路线图

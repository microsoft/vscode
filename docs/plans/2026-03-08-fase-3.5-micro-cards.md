# Fase 3.5: Sandboxed Webview Cards, Bridge API, App Mode

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Transform the empty canvas into a Liquid App where cards are **sandboxed HTML webviews** communicating through a typed bridge API (`window.phonon`). Any developer — React, Vue, vanilla JS, Python+WASM — can build a card without knowing VS Code internals. App Mode hides VS Code chrome so Layer 3 users see only canvas + sidebar + chat.

**Architecture:** Each canvas slot creates an `<iframe sandbox>` (or VS Code `WebviewElement`). The platform injects a bridge script that exposes `window.phonon = { data, navigate, intent, ... }`. Cards communicate with the host via `postMessage`. The host routes data requests to the registry's data providers. Cards are declared in extension `package.json` and point to HTML entry files.

**Tech Stack:** VS Code WebviewElement (sandboxed iframe), postMessage bridge, JSON Schema validation, IWorkbenchLayoutService for App Mode.

**This is NOT a VS Code extension system. Cards are HTML files. The bridge is the API. Any language compiles to what runs in a browser.**

## Card Architecture (read this before implementing)

```
Card (HTML file, any framework)
    |
    | window.phonon.data.fetch('dish', { limit: 10 })
    |
    v
Bridge Script (injected by platform into iframe)
    |
    | postMessage({ type: 'phonon:data:fetch', entity: 'dish', ... })
    |
    v
Canvas Slot Host (LiquidCanvasEditor._handleSlotMessage)
    |
    | routes to ILiquidModuleRegistry data providers
    |
    v
Data Provider (mock for now, Supabase/REST later)
    |
    v
Response flows back: host → postMessage → bridge → card callback
```

A card developer writes:
```html
<!-- cards/FoodCost.html -->
<!DOCTYPE html>
<html>
<head><style>/* card styles */</style></head>
<body>
  <div id="root"></div>
  <script>
    // window.phonon is injected by the platform
    window.phonon.onReady(async () => {
      const dishes = await window.phonon.data.fetch('dish');
      const avgCost = dishes.reduce((s, d) => s + d.foodCostPercent, 0) / dishes.length;
      document.getElementById('root').innerHTML = `
        <h2>${avgCost.toFixed(1)}%</h2>
        <p>Food Cost Medio</p>
      `;
    });
  </script>
</body>
</html>
```

That's it. No VS Code imports. No TypeScript required. No extension API.

```typescript
/** Messages from card (iframe) → host (canvas editor) */
export type CardToHostMessage =
	| { type: 'phonon:data:fetch'; requestId: string; entity: string; query?: Record<string, unknown> }
	| { type: 'phonon:data:mutate'; requestId: string; entity: string; operation: 'create' | 'update' | 'delete'; data: unknown }
	| { type: 'phonon:navigate'; viewId: string; params?: Record<string, unknown> }
	| { type: 'phonon:intent'; description: string }
	| { type: 'phonon:setTitle'; title: string }
	| { type: 'phonon:setLoading'; loading: boolean }
	| { type: 'phonon:ready' };

/** Messages from host (canvas editor) → card (iframe) */
export type HostToCardMessage =
	| { type: 'phonon:data:response'; requestId: string; success: boolean; data?: unknown; error?: string }
	| { type: 'phonon:params'; params: Record<string, unknown> }
	| { type: 'phonon:init'; cardId: string; entity?: string };
```

**Step 2: Create the injectable bridge script**

This is a plain JS file (NOT TypeScript) that gets loaded into the iframe before the card's own code. It provides `window.phonon`:

```javascript
// phonon-card-bridge.js — injected into card iframe by the platform

(function() {
  'use strict';

  const pending = new Map(); // requestId → { resolve, reject }
  let readyCallbacks = [];
  let initialized = false;

  window.phonon = {
    data: {
      fetch(entity, query) {
        return sendRequest('phonon:data:fetch', { entity, query });
      },
      mutate(entity, operation, data) {
        return sendRequest('phonon:data:mutate', { entity, operation, data });
      }
    },
    navigate(viewId, params) {
      window.parent.postMessage({ type: 'phonon:navigate', viewId, params }, '*');
    },
    intent(description) {
      window.parent.postMessage({ type: 'phonon:intent', description }, '*');
    },
    setTitle(title) {
      window.parent.postMessage({ type: 'phonon:setTitle', title }, '*');
    },
    setLoading(loading) {
      window.parent.postMessage({ type: 'phonon:setLoading', loading }, '*');
    },
    onReady(callback) {
      if (initialized) { callback(); }
      else { readyCallbacks.push(callback); }
    },
    params: {}
  };

  function sendRequest(type, payload) {
    return new Promise((resolve, reject) => {
      const requestId = crypto.randomUUID();
      pending.set(requestId, { resolve, reject });
      window.parent.postMessage({ type, requestId, ...payload }, '*');
    });
  }

  window.addEventListener('message', (event) => {
    const msg = event.data;
    if (!msg || !msg.type) return;

    if (msg.type === 'phonon:data:response') {
      const p = pending.get(msg.requestId);
      if (p) {
        pending.delete(msg.requestId);
        msg.success ? p.resolve(msg.data) : p.reject(new Error(msg.error || 'Unknown error'));
      }
    } else if (msg.type === 'phonon:params') {
      window.phonon.params = msg.params || {};
    } else if (msg.type === 'phonon:init') {
      initialized = true;
      readyCallbacks.forEach(cb => cb());
      readyCallbacks = [];
    }
  });

  // Signal readiness to host
  window.parent.postMessage({ type: 'phonon:ready' }, '*');
})();
```

**Step 3: Create host-side message handler**

In `liquidCardBridge.ts`, create `LiquidCardSlotHost`:

```typescript
export class LiquidCardSlotHost extends Disposable {
	constructor(
		private readonly iframe: HTMLIFrameElement,
		private readonly cardId: string,
		private readonly entity: string | undefined,
		private readonly params: Record<string, unknown> | undefined,
		private readonly dataResolver: ILiquidDataResolver,
	) {
		super();
		this._register(dom.addDisposableListener(window, 'message', (e: MessageEvent) => {
			if (e.source !== this.iframe.contentWindow) return;
			this._handleMessage(e.data as CardToHostMessage);
		}));
	}

	private async _handleMessage(msg: CardToHostMessage): Promise<void> {
		switch (msg.type) {
			case 'phonon:ready':
				this.iframe.contentWindow?.postMessage({
					type: 'phonon:init', cardId: this.cardId, entity: this.entity
				}, '*');
				if (this.params) {
					this.iframe.contentWindow?.postMessage({
						type: 'phonon:params', params: this.params
					}, '*');
				}
				break;
			case 'phonon:data:fetch':
				try {
					const data = await this.dataResolver.fetch(msg.entity, msg.query);
					this.iframe.contentWindow?.postMessage({
						type: 'phonon:data:response', requestId: msg.requestId, success: true, data
					}, '*');
				} catch (err) {
					this.iframe.contentWindow?.postMessage({
						type: 'phonon:data:response', requestId: msg.requestId, success: false, error: String(err)
					}, '*');
				}
				break;
			// ... handle other message types
		}
	}
}
```

```typescript
private _renderSlot(parent: HTMLElement, slot: ICompositionSlot): void {
	const slotEl = dom.append(parent, dom.$('.liquid-canvas-slot'));
	const header = dom.append(slotEl, dom.$('.liquid-canvas-slot-header'));
	header.textContent = slot.label ?? slot.cardId ?? slot.viewId ?? 'Unknown';
	const body = dom.append(slotEl, dom.$('.liquid-canvas-slot-body'));

	if (slot.cardId) {
		const card = this.registry.cards.find(c => c.id === slot.cardId);
		if (card) {
			this._renderCardIframe(body, card, slot.params);
			return;
		}
	}

	// Fallback: placeholder for views (future)
	const viewLine = dom.append(body, dom.$('div'));
	viewLine.textContent = `View: ${slot.viewId ?? slot.cardId ?? 'unknown'}`;
}

private _renderCardIframe(
	container: HTMLElement,
	card: ILiquidCard,
	params?: Record<string, unknown>,
): void {
	const iframe = dom.append(container, dom.$('iframe.liquid-card-iframe'));
	iframe.setAttribute('sandbox', 'allow-scripts allow-same-origin');
	iframe.setAttribute('loading', 'lazy');

	// Build the card HTML: bridge script + card entry
	// The bridge script is inlined, then the card HTML is loaded.
	// For MVP we use srcdoc with the bridge + a fetch of the entry HTML.
	// Production would use a proper content security policy.
	const bridgeScriptUri = this._getBridgeScriptContent();
	const cardEntryUrl = card.entryUri.toString(true);

	iframe.srcdoc = `
		<!DOCTYPE html>
		<html>
		<head>
			<meta charset="utf-8">
			<style>
				* { margin: 0; padding: 0; box-sizing: border-box; }
				body { font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif; color: #ccc; background: transparent; padding: 8px; }
			</style>
			<script>${bridgeScriptUri}</script>
		</head>
		<body>
			<div id="phonon-card-root">Loading...</div>
			<script>
				fetch('${cardEntryUrl}')
					.then(r => r.text())
					.then(html => {
						document.getElementById('phonon-card-root').innerHTML = '';
						const range = document.createRange();
						const fragment = range.createContextualFragment(html);
						document.getElementById('phonon-card-root').appendChild(fragment);
					})
					.catch(err => {
						document.getElementById('phonon-card-root').textContent = 'Card load error: ' + err.message;
					});
			</script>
		</body>
		</html>
	`;

	// Create host-side message handler
	const host = new LiquidCardSlotHost(iframe, card.id, card.entity, params, this._dataResolver);
	this._slotHosts.add(host);
}
```

NOTE: The exact iframe loading strategy may need adjustment for VS Code's content security policy. If `srcdoc` doesn't work with fetch, switch to `WebviewElement` (VS Code's built-in sandboxed webview). The architecture stays the same — only the container changes.

```typescript
handleClick(item: ITreeItem): void {
	const node = this._findNode(item.handle, this.registry.sidebarTree);
	if (!node?.view) { return; }

	// Dashboard: compose grid from dashboard-tagged cards
	const dashboardCards = this.registry.cards;
	if (node.view.includes('Dashboard') && dashboardCards.length > 0) {
		const dashCards = dashboardCards.filter(c => c.tags.includes('dashboard'));
		if (dashCards.length > 0) {
			this._onDidRequestNavigation.fire({
				layout: 'grid',
				slots: dashCards.map(c => ({ cardId: c.id, label: c.label })),
				title: node.label,
			});
			return;
		}
	}

	// Regular view
	this._onDidRequestNavigation.fire({
		layout: 'single',
		slots: [{ viewId: node.view }],
		title: node.label,
	});
}
```

## What This Achieves

After all 9 tasks:
- `Cmd+Shift+A` → App Mode: sidebar + canvas + chat, no VS Code chrome
- Click "Dashboard" → 6-card grid, each card is an HTML file in a sandboxed iframe
- Cards fetch real (mock) data through `window.phonon.data.fetch()`
- Ask Claude "mostra food cost e ordini" → AI generates `phonon-intent` → canvas opens 2-card split
- A web developer who knows only HTML/JS can build a new card without touching VS Code source

## What This Does NOT Do (Fase 4+)

- Card SDK npm package (currently bridge is injected, not imported)
- Real data providers (Supabase, REST) — mock only
- Permission model (cards can fetch any entity)
- Marketplace / card distribution
- WASM card support
- Multi-tenant / user management
